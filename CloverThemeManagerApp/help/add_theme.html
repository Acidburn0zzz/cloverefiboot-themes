Clover Theme Manager

I've created a theme. How can I add it?

The following steps show how you can add a new theme to the repository so it automatically becomes available for anyone to install.

1 - Sourceforge user account

You need to have a sourceforge account and know your username. If you don't already have one then you can register by clicking the following link and clicking 'Join' in the top right corner.
http://sourceforge.net

2 - Ask permission to add theme

You will need to ask an admin associated with Clover to add you as a member and assign write access for you and your theme. A list of administrators who can do this can be found in the following access list with 'Admin' beside their username:
https://sourceforge.net/p/cloverefiboot/_members/

The administrator will need to know your sourceforge username and the title of your theme (exact spelling, capitalisation etc.) before they can add you to the access list.

One place to ask is in the Clover Theme Manager topic at projectosx
http://www.projectosx.com/forum/index.php?showtopic=3329

3 - Clone the existing repository

Decide where on your local hard drive you wish to store the Clover theme repository.
Load Terminal.app and cd to that directory.

As an example, here I wish to clone the repository to my HDD partition named Data.

cd /Volumes/Data/

Then enter the following, replacing USERNAME with your sourceforge username:

git clone ssh://USERNAME@git.code.sf.net/p/cloverefiboot/themes cloverefiboot-themes

After pressing return you will be asked for your sourceforge password and then see something like the following:

4 - Add your theme to your local cloned repository

You can add your theme to the path/to/cloverefiboot-themes/themes directory by either copying it using the Terminal or dragging and dropping from within the Finder.

To check all is good, you can cd in to cloverefiboot-themes directory and run git status.
To continue my example from step 3:

cd /Volumes/Data/cloverefiboot-themes
git status

After pressing return you will see something like this:

On branch master
Your branch is up-to-date with 'origin/master'.

Untracked files:
(use "git add ..." to include in what will be committed)

themes/myNewTheme/

nothing added to commit but untracked files present (use "git add" to track)

5 - Stage your change

The previous message tells you there are untracked files. You can now follow the suggestion and git add your theme to git's staging area. To continue my example from above:

git add themes/myNewTheme/

You won't see any feedback after running this command but if you wish to check all is good then you could run git status again:

You can now commit this change with a descriptive message using the git commit command with the -m option.

git commit -m "Adding my new theme."

git push origin master

8 - Complete

Congratulations. You have just added your theme to the repository. The next time you or anybody else launches Clover Theme Manager, your theme will be available for installation.

9 - Keeping your local repo up to date

Other users can make changes to the repository on the server. To keep your local repository in sync with the server, cd in to cloverefiboot-themes directory and run the git pull command.

10 - Making changes to your theme

If you later make a change to your theme files in your local repository and want to update the main repository then follow step 4 from git status onwards to stage, commit and push the changes.
